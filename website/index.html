A systems language for backend work.

Mako compiles .mko source to C and then to native binaries. It has no tracing garbage collector or VM. The authoritative implementation, tests, limits, and release evidence live in GitHub.

What is implemented

Native compilation

The compiler emits C and invokes the target C toolchain. Release builds use -O3 -flto.

No garbage collector

Memory uses scopes, hold, share, arenas, pools, and explicit resource APIs. Removed gc_* calls are compile errors.

Structured concurrency

crew owns kick jobs and cancel-joins them at scope exit. Explicit detach requires detached_join_all().

Backend standard library

Networking, TLS, HTTP, JSON, databases, crypto, files, and observability surfaces are included with platform and optional-library limits documented.

Safety mechanisms

Move checking, bounds checks, typed errors, Send restrictions, secret wiping, and leak diagnostics prevent important classes of backend mistakes.

Cross targets

CI verifies native Linux/macOS/Windows builds plus Linux musl and Windows GNU cross-smoke. Other targets require their own toolchains.

Try Mako

A published binary release can be installed without Rust or Cargo on the install machine. Clang is still needed to compile a user's Mako program.

curl -fsSL https://github.com/loreste/mako/releases/latest/download/install-release.sh | bash
source "$HOME/.local/share/mako/env.sh"
mako version

Check the GitHub releases for supported platform artifacts.
